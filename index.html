The Game

battleMETAL is a successor to the 90's mech simulators of old. A delightful modernization of commanding giant robots bristling with firepower. Players deploying into mechs on the blighted world of Geiserath will be tasked with completing a variety of missions. The plan is to deliver a solid single-player experience with a co-op campaign as a stretch goal.

The Tech

battleMETAL is built upon the Darkplaces Engine which itself is a source port of the classic first person shooter, Quake. The game leverages the extensions of the Darkplaces engine to deliver a unique game experience of giant robot combat.

Being based on the dependable Quake engine and bolstered by the major extensions that Darkplaces provides, battleMETAL can pretty much run on almost any modern system with speed. Networking is also reliable, with the ability to run LAN groups, and host your own servers for the game. The project has space for creating a master server specifically for battleMETAL which will make connecting to battleMETAL game servers easier.

Source Code

battleMETAL is primarily coded in Quake's scripting language, 'QuakeC' (QC). 90% of the original Quake's QC code has been stripped out and replaced with functionality in service to battleMETAL's design goals. Some sections of code have been sourced from tutorials online, but a large majority of this code has been written by the designer. Current Development

The designer has been using a task-tracking website called Trello to guide and track the work being done on the game. You can follow along with this progress by visiting the link below.
A quick primer for understanding the Trello board -
[Monthly] - objectives which are generic directives to guide the work per-month.
[Backlog] - simply stated, its a grab bag of low-level or far-off tasks that aren't super important to achieve now.
[Alpha 2] - specific tasks that need to be completed for the second Alpha.
[New] - tasks that crop up that should be worked on soon.
[Todo] - tasks that will be worked on shortly.
[In Progress] - tasks that are currently being worked on.
[Complete in X] - the rest is a log of what has been accomplished per month going back to when Trello was first used.

Modding battleMETAL

A major design goal of the entire battleMETAL project is to, in addtion to making a fun mech game, deliver an api of sorts for other developers to rapidly stand up their own mech games. Quake, Darkplaces, and the battleMETAL code are all primed for modding. A lot of documentation can be found online for modifying both games, and below are links to documents for modifying battleMETAL.

Below you will find links to main pieces of documentation that the designer has authored as he developed the main project of battleMETAL. battleMETAL makes radical changes to the QC source code and is not compatible with any other mods that currently exist. Maps that are made for the game, although its similar to regular id Tech games, uses its own set of map entities and game logic to create a functional maps.
